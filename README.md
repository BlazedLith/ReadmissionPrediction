# Predicting 30-Day Hospital Readmission in Diabetic Patients
 
A comparative machine learning study on 30-day hospital readmission prediction for diabetic patients. Extends the work of Strack et al. (2014) by comparing four supervised models, adding unsupervised clustering, and performing a formal fairness analysis across racial groups with post-processing bias mitigation.
 
## Dataset
 
UCI Diabetes 130-US Hospitals dataset (Clore et al., 2014). 101,766 patient encounters from 130 US hospitals, 1999-2008. After preprocessing: 69,973 encounters, 32 features, 9% positive class (readmitted within 30 days).
 
DOI: https://doi.org/10.24432/C5230J

## Models
 
| Model | AUC | Recall | Precision | F1 |
|---|---|---|---|---|
| Random Forest | 0.6442 | 0.208 | 0.213 | 0.210 |
| XGBoost | 0.6438 | 0.557 | 0.137 | 0.220 |
| Logistic Regression | 0.6414 | 0.529 | 0.136 | 0.216 |
| Decision Tree | 0.6277 | 0.583 | 0.124 | 0.204 |
| Soft Voting Ensemble | 0.6458 | 0.444 | — | 0.225 |
 
XGBoost is the deployment model. It is within 0.0004 AUC of the best model and catches 55.7% of actual readmissions, more than double what Random Forest catches. In a clinical context, missing a high-risk patient is more costly than a false alarm.

## Running the Notebooks
 
Run notebooks in order. Each notebook saves outputs that the next one depends on.
 
```bash
jupyter notebook
```
 
Order: `01_data_preprocessing` → `02_eda` → `03_modeling` → `04_fairness` → `05_statistical_significance`
 
Note: `03_modeling.ipynb` saves all models and prediction arrays to `models/`. Run it before the statistical significance notebook.
 
## Running the App
 
```bash
streamlit run app.py
```
 
The app runs at `http://localhost:8501` and has six pages: Overview, EDA, Model Performance, Statistical Analysis, Fairness Analysis, and Predict.
 
The Predict page supports both single-patient input and batch CSV upload. A template CSV can be downloaded from inside the app.
